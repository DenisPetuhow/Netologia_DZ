Лямбда-функции (анонимные функции) в C++ — это удобный способ определить функцию прямо в месте использования, не давая ей отдельного имени. В данном коде:

```cpp
#include <iostream>

const auto func = (int val) 
{
    std::cout << val << std::endl;
};

int main()
{
    func(100);
    return 0;
}
```

строчка

```cpp
const auto func = (int val) { std::cout << val << std::endl; };
```

создаёт лямбда-функцию, принимающую один аргумент типа int и выводящую его в консоль. Лямбда присваивается переменной func, которая по сути становится функцией (объектом-функцией). В main она вызывается с параметром 100, что приводит к выводу "100".

Квадратные скобки `` — это захват переменных из окружающей области видимости (scope capture). В данном случае скобки пустые, т.е. никакие переменные не захватываются — лямбда полностью "самодостаточна" и работает только с передаваемыми ей аргументами.

Если написать:

- `` — лямбда захватит все переменные из внешней области видимости по ссылке. Это важно, если внутри лямбды нужно читать или изменять внешние переменные. Например:

    ```cpp int x = 10;
    auto f = () { x = 20; };
    f();
    // Теперь x == 20
    ```

- `` — такой записи в C++ нет. Но `` иногда ошибочно используют вместо ``. `` — захват всех внешних переменных по значению (копированием). В таком случае лямбда получает копии переменных, и если такие переменные внутри изменять, это не повлияет на оригиналы:

    ```cpp int x = 10;
    auto f = () { /* x можно читать, но менять нельзя */ };
    ```

Итак:
- `` — ничего не захватывает (может использовать только параметры)
- `` — захватывает все внешние переменные по ссылке (можно изменять)
- `` — захватывает все внешние переменные по значению (создаются копии)

Лямбды удобны когда функция нужна "здесь и сейчас", их можно передавать как аргументы в другие функции (например, алгоритмы STL) или использовать как обработчики событий.
